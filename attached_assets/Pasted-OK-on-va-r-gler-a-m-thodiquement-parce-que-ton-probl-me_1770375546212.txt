OK, on va régler ça méthodiquement, parce que ton problème est classique avec Daily + Expo/Replit, et ce n’est pas un bug Daily.

👉 Résumé du problème

La vidéo s’affiche bien

Mais :

en paysage

toute petite

pas en plein écran

Tu développes sur Replit avec Expo

1️⃣ La vraie cause (important)

Sur Expo / React Native, Daily n’est PAS responsable du layout.

Daily :

envoie correctement le flux vidéo

respecte l’orientation native de la caméra (souvent paysage)

👉 C’est TON conteneur React Native qui contraint la taille et l’orientation.

2️⃣ Les 3 causes techniques les plus fréquentes (dans ton cas)
❌ Cause n°1 — Le composant vidéo n’a pas flex: 1

Résultat :
➡️ la vidéo prend la taille minimale, donc minuscule.

❌ Cause n°2 — Orientation non verrouillée

Résultat :
➡️ la caméra renvoie du landscape, même si le téléphone est en portrait.

❌ Cause n°3 — resizeMode mal réglé

Résultat :
➡️ la vidéo est “contenue” au lieu de remplir l’écran.

3️⃣ La solution QUI MARCHE (Expo + Daily)
✅ Étape 1 — Forcer le plein écran via le layout

Dans ton écran vidéo (très important) :

import { View, StyleSheet } from "react-native";

export default function VideoScreen() {
  return (
    <View style={styles.container}>
      <DailyVideo />
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: "black",
  },
});


⚠️ Obligatoire :

flex: 1

fond noir (sinon bandes visibles)

✅ Étape 2 — Le composant vidéo Daily en plein écran

Si tu utilises DailyIframe ou un wrapper WebRTC :

<View style={{ flex: 1 }}>
  <RTCView
    streamURL={videoTrack}
    style={{ flex: 1 }}
    objectFit="cover"
  />
</View>


🔑 Le point clé ici :

flex: 1

objectFit="cover" (ou équivalent)

👉 Sans cover, la vidéo reste petite.

4️⃣ Forcer l’orientation portrait (CRUCIAL)

Daily renvoie souvent du paysage par défaut.

Dans Expo :

📦 Installer Screen Orientation
expo install expo-screen-orientation

🔒 Forcer portrait
import * as ScreenOrientation from "expo-screen-orientation";

useEffect(() => {
  ScreenOrientation.lockAsync(
    ScreenOrientation.OrientationLock.PORTRAIT
  );

  return () => {
    ScreenOrientation.unlockAsync();
  };
}, []);


👉 Résultat :

la vidéo s’adapte au téléphone

fini le paysage forcé

5️⃣ Si tu utilises Daily en WebView (cas Replit très courant)

⚠️ IMPORTANT : si tu es en WebView, il faut aussi :

<WebView
  style={{ flex: 1 }}
  allowsFullscreenVideo
  mediaPlaybackRequiresUserAction={false}
/>


Et côté HTML Daily :

<video
  autoplay
  playsinline
  style="
    position: fixed;
    inset: 0;
    width: 100%;
    height: 100%;
    object-fit: cover;
  "
></video>


👉 Sans playsinline, iOS force parfois un mini player.

6️⃣ Checklist rapide (à cocher)

✔ flex: 1 sur TOUS les parents
✔ objectFit: cover
✔ Orientation verrouillée (Expo)
✔ playsinline si WebView
✔ Pas de width: 300 / height: 200 caché
