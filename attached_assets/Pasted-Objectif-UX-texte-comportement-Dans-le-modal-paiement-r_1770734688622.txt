Objectif UX (texte + comportement) :
Dans le modal paiement, remplacer les libellés actuels par :
1) Bouton principal : "Créer mon compte Stripe Connect"
2) Bouton secondaire : "J’ai déjà un compte Stripe Connect"

Comportement obligatoire :
- Les deux boutons doivent ouvrir Stripe Connect Express (URL qui commence par https://connect.stripe.com/express/…)
- On ne doit JAMAIS ouvrir le flux Stripe standard/OAuth (écran email + message "You can only create new accounts…")

Implémentation attendue (Connect Express via Account Links) :

A) BACKEND (/server)
1) Vérifie qu’il existe un endpoint qui crée un compte Express + renvoie un accountLink :
   POST /api/stripe/express/create-and-onboard
   - stripe.accounts.create({ type: "express" })
   - stripe.accountLinks.create({ account: acct.id, type: "account_onboarding", refresh_url, return_url })
   - renvoie JSON: { account_id: acct.id, url: link.url }

2) Vérifie qu’il existe un endpoint qui génère un accountLink pour un compte existant :
   GET /api/stripe/express/account-link?account_id=acct_xxx
   - stripe.accountLinks.create({ account: account_id, type: "account_onboarding", refresh_url, return_url })
   - renvoie JSON: { url: link.url }

B) FRONT (/signtouch-app-main)
1) Dans le composant du modal, remplace uniquement les textes des boutons par :
   - "Créer mon compte Stripe Connect"
   - "J’ai déjà un compte Stripe Connect"

2) Branche les actions :
   - Au clic sur "Créer mon compte Stripe Connect" :
     appeler POST `${EXPO_PUBLIC_STRIPE_SERVER_URL}/api/stripe/express/create-and-onboard`
     récupérer { account_id, url }
     sauvegarder account_id dans le profil célébrité (storage ou DB si déjà en place)
     ouvrir url via Linking.openURL(url)

   - Au clic sur "J’ai déjà un compte Stripe Connect" :
     NE PAS demander d’email.
     1) si un account_id est déjà sauvegardé pour cette célébrité -> appeler
        GET `${EXPO_PUBLIC_STRIPE_SERVER_URL}/api/stripe/express/account-link?account_id=${accountId}`
        ouvrir url
     2) Sinon, afficher un message clair :
        "Aucun compte Stripe Connect n’est associé à ce profil. Cliquez sur 'Créer mon compte Stripe Connect'."
     (Option debug facultative seulement si déjà existante : champ pour saisir acct_... mais masqué derrière un toggle "Mode admin".)

3) Validation obligatoire :
- Après clic sur chaque bouton, confirmer (console log) que l’URL ouverte commence par:
  https://connect.stripe.com/express/

Ne touche pas :
- au webhook sécurisé
- à la logique 85/15
- aux paiements Checkout

Réponds avec :
- fichier(s) modifié(s)
- les 2 libellés mis à jour
- la preuve que l’URL ouverte est bien connect.stripe.com/express
