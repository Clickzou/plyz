A) Côté Expo (TypeScript)
1) Installer et configurer RevenueCat:

B) Côté Supabase (Edge Functions Deno)
1) Edge Function `create_paid_session_intent` (POST)
   - Auth obligatoire (utiliser l’Authorization Bearer JWT Supabase)
   - Input JSON:
     {
       "celebrity_id": "uuid",
       "product_id": "string",
       "duration_minutes": 1|3|5|10,
       "rc_transaction_id": "string",
       "platform": "apple|google",
       "gross_amount_cents": 200..10000,
       "currency": "EUR"
     }
   - Vérifier la transaction via RevenueCat API (secret côté serveur):
     - appeler l’endpoint RevenueCat approprié pour vérifier qu’une transaction existe et appartient à app_user_id
     - refuser si transaction inconnue, déjà utilisée, ou user mismatch
   - Idempotence:
     - si rc_transaction_id existe déjà dans fan_transactions -> renvoyer la session existante (200)
   - Créer en DB (service role):
     - sessions: fan_id=auth.uid, celebrity_id, duration_minutes, status='ready', scheduled_at=NOW()
     - fan_transactions: session_id, fan_id, celebrity_id, platform, product_id, gross_amount_cents, currency,
       rc_transaction_id, status='store_confirmed', store_confirmed_at=NOW()
   - Retourner:
     { "session_id": "uuid", "transaction_id": "uuid" }

2) Edge Function `rc_webhook` (POST)
   - Recevoir les webhooks RevenueCat (purchase/refund/cancel)
   - Vérifier un secret header (WEBHOOK_SECRET)
   - Log idempotent dans table `webhook_events` (unique on event_id)
   - Mettre à jour fan_transactions:
     - purchase -> status='store_confirmed' + store_confirmed_at
     - refund -> status='refunded' + refunded_at
   - Répondre 200 rapidement

C) SQL Supabase à ajouter si manquant

   - source text not null default 'revenuecat'
   - received_at timestamptz default now()
   - payload jsonb not null
   - processed boolean default true
