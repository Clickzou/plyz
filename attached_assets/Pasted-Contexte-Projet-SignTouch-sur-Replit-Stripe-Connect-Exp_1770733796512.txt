Contexte :
- Projet SignTouch sur Replit
- Stripe Connect Express est déjà utilisé (comptes connectés acct_..., webhooks account.updated OK)
- Mais dans l’app, quand on clique sur "Recevez vos paiements" / "J’ai déjà un compte Stripe", Stripe affiche :
  "You can only create new accounts if you've signed up for Connect…"
=> Cela signifie que l’app ouvre le mauvais flux (standard/OAuth) au lieu de l’onboarding Express.

OBJECTIF UNIQUE :
Remplacer ce flux par le flux correct Stripe Connect Express :
- Générer un "Account Link" côté backend (stripe.accountLinks.create)
- Ouvrir ce lien dans l’app
- Ne plus afficher l’écran Stripe "Créer mon compte / J’ai déjà un compte Stripe" (qui est le mauvais flux).

INSTRUCTIONS PRÉCISES :

A) BACKEND (dans /server)
1) Ouvre `server/index.js` (ou le fichier qui gère Stripe).
2) Ajoute un endpoint qui génère un accountLink Express :

   GET /api/stripe/express/account-link?account_id=acct_xxx

   - vérifier que account_id est présent
   - créer un lien via stripe.accountLinks.create({
       account: account_id,
       type: "account_onboarding",
       refresh_url: <une URL de retour>,
       return_url: <une URL de retour>
     })
   - renvoyer JSON : { url: link.url }

3) Utilise les URLs suivantes (simples, côté web) :
   refresh_url = "https://3aa55d0d-178c-4720-bdee-f8cea294f71b-00-3sqzk84ygwh7z.picard.replit.dev/stripe/refresh"
