PROMPT REPLIT — SUPABASE EDGE FUNCTION joinEvent (ZÉRO REALTIME)

OBJECTIF
Créer une Supabase Edge Function nommée `joinEvent` qui permet à un fan de rejoindre une session événement via `join_code`, tout en LIMITANT de façon fiable le nombre de viewers (sans Realtime).
La fonction doit être ATOMIQUE (pas de race condition) : nettoyage des viewers expirés + comptage + insertion/upsert dans une seule transaction.

STACK
- Supabase Edge Functions (Deno / TypeScript)
- Supabase Postgres
- App Expo/React Native côté client

PRÉREQUIS DB (à créer si absent)
1) Table event_sessions
- id uuid pk default gen_random_uuid()
- title text not null
- starts_at timestamptz not null
- ends_at timestamptz not null
- status text not null default 'live'  -- 'scheduled'|'live'|'ended'
- join_code text unique not null
- viewer_soft_limit int default 5000
- created_by uuid

2) Table event_viewers
- id uuid pk default gen_random_uuid()
- event_id uuid not null references event_sessions(id) on delete cascade
- viewer_id text not null
- last_seen_at timestamptz not null default now()
- created_at timestamptz not null default now()
UNIQUE(event_id, viewer_id)

Index:
- create index on event_viewers(event_id, last_seen_at desc)
- create index on event_sessions(join_code)

RLS
- L’edge function utilisera le Service Role pour bypass RLS.
- Le client n’a PAS besoin d’insert direct dans event_viewers.

SPÉCIFICATIONS API
Route: POST /functions/v1/joinEvent
Body JSON:
{
  "join_code": "ABC123",
  "viewer_id": "user-uuid-or-device-id",
  "client_ts": "optional ISO date"
}

Réponse JSON succès:
{
  "allowed": true,
  "event": {
    "id": "...",
    "title": "...",
    "status": "live",
    "ends_at": "...",
    "viewer_soft_limit": 5000
  }
}

Réponse JSON refus:
{
  "allowed": false,
  "reason": "full" | "not_found" | "ended" | "not_live" | "bad_request",
  "message": "..."
}

RÈGLES MÉTIER joinEvent
1) Trouver l’événement via join_code
2) Refuser si event.status != 'live' OU ends_at < now()
3) Nettoyer les viewers expirés : last_seen_at < now() - 2 minutes (TTL=120s)
4) Compter viewers actifs (après cleanup)
5) Si count >= viewer_soft_limit => refuser (allowed=false, reason=full)
6) Sinon upsert viewer (event_id, viewer_id) en mettant last_seen_at=now()
7) Retourner allowed=true + infos event

IMPORTANT: ATOMICITÉ
- Toute la logique doit être faite en 1 transaction SQL.
- Utiliser une fonction SQL (RPC) pour garantir atomicité, puis l’appeler depuis l’Edge Function.
=> Donc : créer une fonction Postgres `join_event(join_code text, viewer_id text)` qui renvoie (allowed bool, reason text, event jsonb).
